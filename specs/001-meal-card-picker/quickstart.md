# 快速開始：餐點抽卡網站

本文件說明如何在本機啟動、驗證與操作餐點抽卡網站的主要流程。

---

## 1. 先決條件

- 已安裝 .NET 10 SDK
- 作業系統可為 macOS、Windows 或 Linux
- 使用桌面瀏覽器（Chrome、Firefox、Safari、Edge）

---

## 2. 啟動專案

在 repo root 執行：

```bash
dotnet build
dotnet run --project CardPicker/CardPicker.csproj
```

預設會啟動本機網站；實際 URL 以終端機輸出為準。

---

## 3. 首次執行預期結果

首次啟動且 `CardPicker/data/cards.json` 尚不存在時，系統應：

1. 建立 `CardPicker/data/` 目錄
2. 建立 `cards.json`
3. 預載數張種子卡牌，且至少各包含：
   - 1 張早餐卡牌
   - 1 張午餐卡牌
   - 1 張晚餐卡牌

若 `cards.json` 已存在，系統不得覆寫既有資料。

---

## 4. 驗證主要使用流程

### 4.1 首頁抽卡

1. 開啟首頁 `/`
2. 選擇早餐、午餐或晚餐
3. 點擊「抽一張」
4. 確認畫面顯示：
   - 卡牌名稱
   - 餐別
   - 完整描述

### 4.2 空卡池提示

1. 刪除某一餐別的所有卡牌
2. 回到首頁選擇該餐別抽卡
3. 確認畫面顯示「目前沒有可抽取的餐點」類型訊息
4. 確認未顯示任何虛假卡牌結果

### 4.3 卡牌列表與搜尋

1. 開啟 `/Cards`
2. 確認預設列表只顯示名稱與餐別摘要
3. 以名稱關鍵字搜尋，確認採大小寫不敏感部分比對
4. 加上餐別條件後，確認僅顯示同時符合條件的卡牌
5. 輸入無符合資料的條件，確認顯示查無結果訊息

### 4.4 新增卡牌

1. 進入 `/Cards/Create`
2. 填入名稱、餐別、描述
3. 送出表單
4. 回到 `/Cards` 後確認新卡牌已存在
5. 回首頁抽卡，確認新卡牌可被抽出

### 4.5 編輯卡牌

1. 進入 `/Cards/Edit?id={cardId}`
2. 更新名稱、餐別或描述
3. 送出後回列表確認資料已更新
4. 再次搜尋或抽卡，確認更新內容已生效

### 4.6 刪除卡牌

1. 進入 `/Cards/Delete?id={cardId}`
2. 明確確認刪除
3. 回列表後確認卡牌已消失
4. 再次搜尋與抽卡，確認該卡牌不再出現

---

## 5. 驗證資料持久化

1. 完成新增、編輯或刪除操作
2. 關閉應用程式
3. 重新執行：

```bash
dotnet run --project CardPicker/CardPicker.csproj
```

4. 回到網站確認先前變更仍然存在

---

## 6. 測試執行方式（功能實作完成後）

```bash
dotnet test
```

預期至少包含：

- 抽卡公平性與空卡池的單元測試
- 搜尋與重複卡牌規則的單元測試
- Razor Pages 表單與 JSON 持久化的整合測試

---

## 7. 檔案與日誌位置

| 類型 | 位置 |
|------|------|
| JSON 卡牌資料 | `CardPicker/data/cards.json` |
| 應用程式設定 | `CardPicker/appsettings.json` |
| 頁面與 PageModels | `CardPicker/Pages/` |
| 服務層 | `CardPicker/Services/` |
| 結構化日誌 | `CardPicker/logs/` |

---

## 8. 常見驗收重點

- 首頁未選擇餐別時不得抽卡
- 同內容卡牌不得重複新增
- 刪除後不得再出現在搜尋與抽卡結果中
- 使用者可見訊息與驗證提示必須是繁體中文
- 正式環境仍須保留 HTTPS / HSTS / Anti-Forgery 保護
